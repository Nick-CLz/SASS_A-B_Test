What it is

Mallard is an experimentation platform for data-science teams at scale. Its headline feature is a set of AI agents that take over the repetitive parts of the data scientist's job — designing experiments, monitoring their health, running the statistics, and writing the ship / no-ship readout — while a human approves the decisions. The agents are trustworthy because they call a deterministic statistics engine as tools and never invent numbers: every figure is traceable to a tool call.

Built for two audiences, one codebase. A rigorous, privacy-first portfolio / interview piece and a sellable SaaS for growth/DS teams. The same engine + agents serve both; only the emphasis of the demo and collateral differ.

The four pillars

🤖 AI-native

Agents design, monitor, analyze & write up experiments — grounded in a real stats engine, fully audited.

🔒 Privacy-first

Pseudonymous units, no PII in the pipeline, a PII guard at ingestion, and a self-host / warehouse-native option.

📊 Statistically serious

Frequentist + Bayesian + sequential ("peek any time"), CUPED variance reduction, SRM detection, power analysis.

🏬 Warehouse-native

DuckDB embedded for the demo; the same SQL runs in Snowflake / BigQuery / Databricks so raw data never leaves.

Why the statistics matter

Gets right what cheap tools get wrong

  • Delta method for ratio metrics & relative-lift CIs (the #1 silent false-positive source).
  • CUPED variance reduction — tighter CIs, same users.
  • Sequential / always-valid inference so peeking is safe by default.
  • SRM (sample-ratio-mismatch) detection — catch the data bug before the readout.

Trustworthy by construction

  • Stats engine is pure & consumes only sufficient statistics → unit-tested vs textbook fixtures.
  • A/A calibration tests (uniform p-values, ≈α false-positive rate) in CI.
  • Assignment is deterministic with cross-language golden fixtures shared by server & SDKs.
  • Every agent number traces to a recorded tool call (no hallucinated p-values).

The AI agents (the differentiator)

Designer

Turns a hypothesis into a sound, powered experiment — picks unit, OEC + guardrails, and a tool-computed sample size.

Health Monitor

Watches a running experiment for SRM, guardrail breaches & anomalies; recommends (never auto-executes) a pause.

Analyst

Runs the correct test per metric, checks SRM first, applies corrections, interprets with honest caveats.

Readout Writer

Writes a senior-DS-quality ship / no-ship readout — sourced to result rows, ready for stakeholders.

How it's being built (the cost play)

An expensive model did the high-value work once — the architecture, the statistics design, and a set of 16 self-contained build prompts (P01–P16) with explicit acceptance criteria. A cheaper model then executes them one at a time. Correctness-critical prompts (assignment, statistics) get a bigger model; boilerplate gets a small one. Each prompt ends green (lint + types + tests) and lands in a PR.
